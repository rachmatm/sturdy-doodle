| 2026-06-12 | Add **vitest** (dev-dep) + `npm test`/`test:watch`; first automated suite is `src/lib/ai.test.ts` covering the provider×key fallback with a mocked `fetch` | The 4-key rollover (2 pixazo + 2 mistral) was only ever proven manually/by-construction, and a live 429-rollover can't be triggered on demand without burning free-tier quota. The fallback is pure deterministic logic, so mocked-`fetch` unit tests verify it in ~250ms, re-run free forever, and need no API keys — far cheaper than re-deriving a manual QA script each pass. Live auth/generation is already proven (2026-06-12 browser QA), so no live smoke is gated on this |
| 2026-06-12 | Surface (not fix) the `isConfigured`/order footgun the test caught: keys for a provider absent from `IMAGE_PROVIDER` are ignored, and the order defaults to `mistral` only | A pixazo-only deployment that omits `IMAGE_PROVIDER=pixazo` reports unconfigured and never uses its keys. The current `.env.local` sets `IMAGE_PROVIDER=pixazo,mistral`, so it's unaffected; documented via a regression test + `.env.example` already shows the var. Changing the default order is a product call, deferred |
| 2026-06-13 | Persist the auto-created Mistral agent id in the DB (new `mistral_agents` table, keyed by a SHA-256 fingerprint of the API key) and reuse it across restarts: `ensureAgent` now does env override → in-memory cache → DB lookup (verified live via `GET /v1/agents`) → create+persist | Previously each process created a fresh agent on first use, accumulating duplicate `logo-generator` agents on the Mistral account across restarts/deploys. Persisting + verifying reuses one agent per key. Fingerprint (not the raw key) keeps the secret out of the DB. DB I/O is best-effort (logged, non-fatal) and a *failed* existence check reuses the stored id rather than spawning a duplicate, so an agent-store hiccup never breaks image generation. Covered by 4 new `ai.test.ts` cases (reuse / create+persist / recreate-when-gone / env-override-skips-store) |
| 2026-06-13 | Add an opt-in `IMAGE_KEY_STRATEGY=round-robin` (default `fallback`): round-robin left-rotates the provider order **and** each provider's key pool by a per-process counter advanced once per request; fallback keeps the strict-priority chain | `POST /api/generate` fans out ~12 image calls that all started on the same first key, so one free-tier key absorbed the burst and 429'd while others sat idle (the "~4 of 12 succeeded" finding). Rotating the starting provider+key spreads the burst. Kept opt-in (default unchanged) and preserved full rollover — every provider×key is still tried before failing, only the starting point rotates (single-key/provider deploy is a no-op). State is a per-process counter (sufficient to spread one instance's fan-out; not shared cross-process). Encapsulated entirely in `ai.ts`; covered by 4 new `ai.test.ts` cases |
| 2026-06-13 | Add **Cloudflare Turnstile** bot protection on `POST /api/generate` + `POST /api/refine` — **beyond the documented MVP scope** (not in PRD/acceptance-criteria); user-requested. New server-only `lib/turnstile.ts` (`verifyTurnstile`/`isTurnstileEnabled`/`clientIp`) + client `TurnstileWidget.tsx`, wired via `LogoStudio` (one shared widget feeds both flows; token sent per request, reset after — tokens are single-use) | The two anonymous AI-cost endpoints can be hammered by bots to burn free-tier image quota (no auth by MVP design, PRD §5.2). **Enforcement is opt-in**: verified only when `TURNSTILE_SECRET_KEY` is set, else skipped with a one-time warn — so local dev/key-less deploys keep working and protection switches on with the secret (matches the codebase's graceful env toggles). The **site key is public** (ships in HTML → `NEXT_PUBLIC_TURNSTILE_SITE_KEY`, given key baked as default); the **secret key is server-only** (`TURNSTILE_SECRET_KEY`, never committed — lives in gitignored `.env.local`). A failed/missing token → existing `INVALID_REQUEST` (already retryable in `ErrorBanner`), so no new error code. Client is **non-blocking** (buttons not gated on token) so a widget that fails to load can't brick the working app; the server is the gate. 9 new `turnstile.test.ts` cases (mocked fetch: skip-when-unset / success / missing-token / failure / network-error / clientIp) |
